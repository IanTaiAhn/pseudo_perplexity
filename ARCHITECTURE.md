# Architecture decisions

> Record every non-obvious decision here as you build. This becomes your interview answer to "walk me through how you built this."

| Date | Decision | Alternatives considered | Reason |
|---|---|---|---|
| 2026-06-29 | Fixed-size chunking (512 tokens, 50 overlap) | Semantic chunking, recursive | Simplest to implement and debug; spec says measure before switching |
| 2026-06-29 | ~~BAAI/bge-small-en-v1.5 for embeddings~~ — superseded 2026-07-02 | OpenAI embeddings, all-MiniLM-L6-v2 | Free, fast, strong MTEB scores; no API cost during dev |
| 2026-06-29 | ChromaDB with cosine distance | FAISS, pgvector, Qdrant | Zero-config local dev; spec mandates Qdrant for prod migration |
| 2026-06-29 | Claude claude-sonnet-4-6 as LLM | GPT-4o | Spec-mandated primary. No OpenAI fallback was ever built — see 2026-07-02 row on the local-LLM dev path instead |
| 2026-06-29 | filter_cited_chunks over extract_citations in response | return all retrieved chunks as citations | Only returning chunks the LLM actually cited reduces noise and false citations |
| 2026-06-29 | Confidence gate at reranker score < 0.3 | Always call LLM | Avoids hallucination when retrieval finds nothing relevant; per spec section 10 |
| 2026-07-01 | Score web chunks via cosine similarity (query embedding · chunk embedding) instead of an in-memory Chroma collection | Ephemeral Chroma collection per query, as literally described in spec Layer 2 step 4 | Same embedding model + normalized vectors make dot product mathematically identical to Chroma's cosine score, so doc and web scores stay comparable; skips the overhead of standing up a collection per query and is easier to unit test |
| 2026-07-02 | Layer 3: hybrid (dense+BM25) retrieval replaces dense-only inside `retrieval_agent.retrieve_from_documents`; cross-encoder reranker now runs in `orchestrator.retrieve` on the merged doc+web pool, replacing the plain score sort | Rerank only the document path, before merging with web results | Web results and doc results were never on a comparable scale even after hybrid fusion (hybrid_score vs. raw cosine); reranking the merged pool gives one consistent, joint-relevance ranking across both sources instead of two differently-scaled sorts |
| 2026-07-02 | `Chunk.score` stays as the single "authoritative current-stage" field (unchanged consumers: confidence threshold, citations, sorting); added `dense_score`/`bm25_score`/`hybrid_score`/`rerank_score` as parallel debug fields instead of replacing `score` | Rename `score` to something stage-specific everywhere it's used | Additive-only change — no existing call site (`query.py` confidence gate, `citation_tracker.py`, tests) needed to change, while still exposing per-stage scores for retrieval-failure debugging |
| 2026-07-02 | `sentence-transformers/all-MiniLM-L6-v2` as the actual default embedding model (`EMBEDDING_MODEL` env var) | `BAAI/bge-small-en-v1.5` (original default) | Smaller/faster to download for local dev; `bge-small-en-v1.5` is still fully supported (`embedder.py` applies its required query-prefix) but is no longer the default |
| 2026-07-02 | Anthropic's own tokenizer (`tokenizers` lib + `anthropic`'s bundled `tokenizer.json`) for chunk token-boundary counting in `chunker.py` | `tiktoken` (originally planned; still listed in requirements.txt but unused) | Chunks are synthesized with Claude, so counting tokens the way Claude sees them is more accurate than counting them the way GPT models see them |
| 2026-07-02 | Optional local HuggingFace LLM via `LOCAL_LLM_MODEL` (e.g. `Qwen/Qwen2.5-1.5B-Instruct`) for zero-cost dev in `generator.py` | OpenAI GPT-4o fallback (original plan) | No OpenAI integration was built; a local HF model gives free, offline dev iteration instead |
| 2026-07-02 | Debug-only routes `/debug/collection` and `/debug/query` added directly in `api/main.py` | Dedicated debug route module, or none | Fast way to inspect ChromaDB contents and raw scores against the confidence gate; intentionally outside the documented `/query`/`/ingest` API surface |
| 2026-07-07 | Layer 4: `ragas_runner.py` wraps Claude via `langchain-anthropic` as the RAGAS judge LLM, and the project's own `sentence-transformers` model via `langchain-huggingface` as the judge embedder | RAGAS's OpenAI defaults | Keeps eval on the same "no OpenAI" path as the rest of the system |
| 2026-07-07 | Pinned `langchain-community==0.3.15`, `anthropic==0.38.0`, `langchain-anthropic==0.2.4`, `pydantic==2.9.2` (up from 2.7.1) for the new RAGAS/langchain/mlflow stack | Latest versions of each package | `ragas==0.2.15` unconditionally imports `langchain_community.chat_models.vertexai`, removed in `langchain-community>=0.4` — hard import failure otherwise. `anthropic>=0.39.0` drops the bundled `tokenizer.json` that `chunker.py` loads at import time, so `anthropic` is pinned at the last version that ships it (0.38.0), which forces `langchain-anthropic` down to 0.2.4 (its `anthropic<1` floor is the last one compatible with 0.38.0). `pydantic` bump is solely to satisfy `langchain`'s `>=2.7.4` floor. No application code changed — `messages.create()` and `BaseModel` usage are unaffected across these ranges. Full resolution verified with `pip check` in a clean venv before pinning. |
| 2026-07-07 | `eval_gate.yml` invokes `python -m evaluation.regression_test`, not `python evaluation/regression_test.py` | The path form (matches the spec's literal wording in section 11) | The path form puts `evaluation/` itself on `sys.path` instead of the repo root, so `from evaluation.ragas_runner import run` (and every `from agents...`/`from api...` absolute import inside it) raises `ModuleNotFoundError`. Module form (`-m`) puts the repo root on `sys.path`, matching how every other absolute import in the codebase already assumes it's run |
